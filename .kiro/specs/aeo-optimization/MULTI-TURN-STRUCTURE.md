# Multi-Turn Conversation Structure

## Overview

CoffieCalm's content is structured to support multi-turn conversations with AI assistants, enabling progressive disclosure and natural follow-up questions.

## Content Hierarchy

### Level 1: Overview (First Question)
**Q: "What is CoffieCalm?"**
- **Hero Section**: Clear entity definition in first 100 words
- **WhatIsCoffieCalm Component**: Semantic relationships and basic understanding
- **Answer**: High-level definition and primary purpose

### Level 2: Details (Follow-up Questions)
**Q: "How does it work?"**
- **HowItWorks Component**: 4-step process for getting started
- **SnippetOptimized Component**: Detailed "How to Get Started" section
- **Answer**: Process explanation with key features

**Q: "What features does it have?"**
- **Features Component**: 6 core features with mental health benefits
- **SnippetOptimized Component**: "Best Features of CoffieCalm" list
- **Answer**: Feature descriptions with benefit connections

### Level 3: Comparisons (Deep Dive)
**Q: "How is it different from therapy?"**
- **Comparison Component**: Detailed CoffieCalm vs. Therapy table
- **SnippetOptimized Component**: "Peer Support vs Professional Therapy" table
- **Answer**: Detailed comparison and use cases

**Q: "Why should I use peer support?"**
- **WhyPeerSupport Component**: Evidence-based benefits and science
- **TopicalAuthority Component**: Deep dive into peer support concepts
- **Answer**: Research-backed benefits and explanations

### Level 4: Specifics (Detailed Questions)
**Q: "Is it safe and private?"**
- **SafetyAndPrivacy Component**: Comprehensive privacy and security information
- **Answer**: Detailed data practices, safety measures, and limitations

**Q: "When should I use it?"**
- **SnippetOptimized Component**: "When to Use CoffieCalm" numbered list
- **ConversationalQA Component**: Specific use case answers
- **Answer**: 5 specific scenarios with explanations

**Q: "What can I talk about?"**
- **TopicalAuthority Component**: Mental wellness concepts and topics
- **VoiceOptimizedFAQ Component**: Common questions about usage
- **Answer**: Comprehensive topic coverage

## Progressive Information Flow

### General → Specific Structure

1. **Hero** (10 seconds)
   - Entity definition
   - Primary value proposition
   - Key metrics

2. **WhatIsCoffieCalm** (30 seconds)
   - Semantic relationships
   - What it is, provides, helps with
   - Target audience

3. **Stats & Testimonials** (1 minute)
   - Social proof
   - User experiences
   - Credibility signals

4. **Features** (2 minutes)
   - Core functionality
   - Feature-benefit connections
   - Mental health outcomes

5. **About & WhyPeerSupport** (3 minutes)
   - Evidence-based content
   - Authority establishment
   - Scientific backing

6. **SnippetOptimized** (4 minutes)
   - Detailed comparisons
   - Process explanations
   - Feature lists

7. **TopicalAuthority** (5 minutes)
   - Deep topic exploration
   - Subtopic coverage
   - Educational content

8. **ConversationalQA** (6 minutes)
   - Natural language queries
   - Specific question answers
   - Use case scenarios

9. **Comparison** (7 minutes)
   - Detailed vs. therapy comparison
   - When to use each
   - Honest limitations

10. **FAQ & VoiceOptimizedFAQ** (8 minutes)
    - Common questions
    - Quick answers
    - Voice-friendly format

11. **SafetyAndPrivacy** (9 minutes)
    - Privacy protections
    - Safety measures
    - Crisis limitations

## Follow-Up Question Patterns

### Pattern 1: What → How
- **Initial**: "What is CoffieCalm?"
- **Follow-up**: "How does it work?"
- **Content**: HowItWorks → SnippetOptimized (Getting Started)

### Pattern 2: How → Why
- **Initial**: "How does peer support work?"
- **Follow-up**: "Why should I use peer support?"
- **Content**: WhyPeerSupport → TopicalAuthority (Peer Support)

### Pattern 3: Why → When
- **Initial**: "Why use CoffieCalm?"
- **Follow-up**: "When should I use it?"
- **Content**: SnippetOptimized (When to Use) → ConversationalQA

### Pattern 4: When → What About
- **Initial**: "When should I use CoffieCalm?"
- **Follow-up**: "What about privacy?"
- **Content**: SafetyAndPrivacy → VoiceOptimizedFAQ

### Pattern 5: General → Specific
- **Initial**: "Tell me about mental wellness"
- **Follow-up**: "How does anonymity help?"
- **Content**: TopicalAuthority (Mental Wellness) → TopicalAuthority (Anonymity)

## Dual-Level Explanations

### High-Level + Detailed Pattern

**Peer Support:**
- **High-level** (WhatIsCoffieCalm): "Peer support connects you with empathetic listeners"
- **Detailed** (TopicalAuthority): 200+ word explanation with subtopics, research, benefits

**Anonymity:**
- **High-level** (Hero): "100% Free & Anonymous"
- **Detailed** (SafetyAndPrivacy): Comprehensive privacy protections, data practices, security measures

**Mental Wellness:**
- **High-level** (Features): "Mental Health Benefit" for each feature
- **Detailed** (TopicalAuthority): 200+ word exploration of mental wellness concepts

**Getting Started:**
- **High-level** (HowItWorks): 4-step overview
- **Detailed** (SnippetOptimized): Detailed numbered process with explanations
